
<b>엔터</b>, 다시 말해 <b>개★행☆문★자</b>는 윈도우(Windows)에서는 CRLF(Carriage Return Line Feed, 캐리지 리턴 라인 피드)로 저장된다.<br>반면에, 유닉스(Unix) 계열의 시스템들에서는 CR(Carriage Return, 캐리지 리턴)로 처리된다.<br>따라서 윈도우로 작성한 텍스트 문서를 리눅스(Linux)에서 열었을 때 각 줄의 끝에 기묘한 <b>^M</b> 이라는 문자들이 붙어있는 짜증나는 모습을 확인할 수 있다.<br><b>하지만 걱정할 것 없다.</b><br>유닉스 시스템(Unix System)의 기본<del>이자 궁극의</del> 에디터인 <b>vi</b>로 문서를 연 다음, <b>:%s/^M//g</b>를 입력해주면 된다.<br>이때, <b>^M</b>은 Ctrl+V+M을 눌러 입력한다.<br>이후 <b>:wq</b>를 입력해 저장하면 문제없이 위키를 이어서 작성할 수 있다. <del>여기까지의 내용은 컴퓨터 사용자라면 알아야 될 기본 상식이다.</del><br>하지만 우리의 최첨단 전곽위키는 위의 복잡한 과정 없이 그저 <b>Enter</b>를 누르면 끝이니 안심하라구^^<br><br>라고 정현재가 썼다. 그러므로 모든 전곽인들은 해당 내용을 숙지하여 <b>CAMEO</b> 수업 때 꼭 아는 척을 하여, 선배들의 마음을 움직이길 바란다.
